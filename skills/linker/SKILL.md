# Thought Linker — 关联网络

## 核心理念

> **关联不是因果链。是叠加态之间的维度相交检测。**
>
> 一个洞察和另一个条目共享某个维度 → 它们关联。
> 就像衣服关联了人（载体）、剪裁（过程）、展示（功能）——都在"衣服"这个维度上。

## 相交维度

| 维度 | 说明 |
|:-----|:-----|
| 载体 | 谁/什么承载了它？ |
| 功能 | 它有什么用？ |
| 过程 | 它是怎么产生的？ |
| 属性 | 它有什么特征？ |
| 前提 | 它建立在什么假设上？ |
| 结论 | 它推出了什么？ |
| 矛盾 | 它和什么有张力？ |

## 输入 → 输出

```
输入: ctx.stages.mapper.all_tags[] + 思想体系所有条目
输出: {
  links: [
    {target_file: "路径", dimension: "载体|功能|过程|...", reason: "..."}
  ],
  is_new_domain: false
}
```

## 操作方式

小皮读取思想体系所有条目标题 → 与当前洞察做维度比对 → 建立关联。

> 提示词核心：
> "这两个条目共享哪个维度？载体/功能/过程/属性/前提/结论/矛盾？
> 只要有一个维度相交，就建立关联。"
